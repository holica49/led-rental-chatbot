### 🧠 고도화된 자연어 파싱 아키텍처
- **결정**: `AdvancedCalendarParser` 클래스 기반 모듈형 파싱 시스템 채택
- **이유**:
  - 복잡한 자연어 처리 로직의 체계적 관리
  - 각 정보 유형별 독립적 파싱 모듈
  - 신뢰도 기반 품질 관리 시스템
  - 확장성과 유지보수성 확보
- **구현**:
  - 날짜/시간 파싱: 상대적/절대적 날짜 모두 지원
  - 참석자 인식: 이름 + 직급 패턴 매칭
  - 회의 유형 분류: 키워드 기반 자동 분류
  - 우선순위 판단: 감정 단어 분석
  - 신뢰도 측정: 추출된 정보량 기반 0-1.0 점수
- **영향**: 
  - 85% 이상 파싱 정확도 달성
  - 한 문장으로 완전한 일정 정보 추출 가능
  - 사용자 만족도 크게 향상

### 🎯 LINE WORKS Calendar API 완전 통합 전략
- **결정**: 기본 캘린더 API + 모든 지원 속성 활용
- **이유**:
  - 별도 캘린더 ID 불필요로 단순성 확보
  - LINE WORKS 문서의 모든 기능 최대 활용
  - 참석자, 알림, 우선순위 등 풍부한 메타데이터 지원
- **API 구조**:
  ```
  POST /users/{userId}/calendar/events
  {
    "eventComponents": [
      {
        "summary": "🔴 🏢 프로젝트 회의 (1명)",
        "location": "강남 스타벅스",
        "attendees": [{"email": "김대리@anyractive.co.kr"}],
        "priority": 1,
        "visibility": "PUBLIC",
        "reminders": [{"trigger": "-PT30M"}]
      }
    ]
  }
  ```
- **영향**:
  - 풍부한 캘린더 이벤트 생성
  - 참석자 자동 초대 기능
  - 우선순위 기반 스마트 설정

### 🤖 MCP (Model Context Protocol) 통합 방식

  - Claude의 강력한 자연어 이해력 활용

### 📊 신뢰도 기반 품질 관리 시스템
- **결정**: AI 파싱 신뢰도 0-100% 측정 및 단계별 처리
- **이유**:
  - 잘못된 일정 등록 방지
  - 사용자 확신도 제공
  - 점진적 품질 개선 가능
- **임계값**:
  - 30% 미만: 재입력 요청
  - 30-70%: 확인 후 등록
  - 70% 이상: 즉시 등록 (권장)
- **계산 방식**:
  - 기본 정보 (날짜, 시간, 제목): 60%
  - 추가 정보 (장소, 참석자, 우선순위): 각 10%
  - 고급 정보 (준비물, 반복): 각 5%
- **영향**:
  - 파싱 품질의 정량적 측정
  - 사용자 신뢰도 향상
  - 점진적 AI 학습 가능

- **결정**: 모듈당 168x168px로 계산
- **이유**:
  - 실제 LED 모듈 스펙과 일치
  - 기존 128px 계산은 부정확했음
  - 고객에게 정확한 해상도 정보 제공
- **계산식**: `(가로모듈수 × 168) x (세로모듈수 × 168)`

### 운반비 구간별 계산
- **결정**: 200개/400개/401개 구간으로 세분화
- **이유**:
  - 트럭 적재량에 따른 실제 비용 반영
  - 단순 고정비에서 합리적 가격 정책으로 전환
  - 대량 주문 시 운반 효율성 고려
- **가격**:
  - 200개 이하: 200,000원 (1톤 트럭)
  - 201-400개: 500,000원 (2.5톤 트럭)
  - 401개 이상: 700,000원 (5톤 트럭)

### 렌탈 실내 견적 간소화

- **영향**: 
  - 렌탈 실내 견적 투명성 향상
